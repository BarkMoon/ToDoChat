# CLAUDE.md

## コミットの運用

- **ユーザーの明示的な指示がない限り、`git commit`（およびステージング目的以外の変更の確定）を行わない。** 変更内容はユーザーが自分で確認するため、実装・編集が完了しても勝手にコミットせず、コミットしてよいか指示を待つ。
- コミットを行う場合も、`git push` はユーザーの明示的な指示がある時だけ行う。

## コミットメッセージのスタイル

このリポジトリでは、過去のコミット履歴に倣い、以下の形式で統一する。

- 1行目: 日本語のタイトル。「〜を追加。」「〜を更新（詳細）。」のように体言止めではなく「。」で終える文で、変更内容を簡潔に要約する
- 本文（変更点が複数・説明が必要な場合のみ）: 空行を挟み、`- ` で始まる箇条書きで要点を列挙する
- モデル名: `(モデル名)` の形式でそのコミットを行ったモデル名を記載する（例: `(Claude Opus 4.8)`、複数モデルが関与した場合は `(Claude Sonnet 5 + Opus 4.8)`）
  - 本文（箇条書き）がある場合: 空行を挟んで末尾の行に単独で書く
  - タイトルのみ・1行で収まる場合: 改行せず、タイトル文の直後に半角スペースを置いて同じ行に書く
- `Co-Authored-By:` トレーラーは使用しない

### 例（本文ありの場合）

```
フルログ復元モードを追加（--resumeで前回の会話全体を再開）。

- session_idを.todochat/sessions.jsonに毎ターン永続化(トグルとは独立)
- グローバルトグル「📜 フルログ復元」ON時は起動時の第一声を要約メモではなく--resumeで会話全体を復元して再開
- 無効なsession_idでは通常起動へ自動フォールバックしUIに通知。/clear・プロジェクト削除でsessions.jsonも後始末
- バックエンドのユニット検証と実CLIでのE2E検証(複数回のアプリ再起動を挟んだ文脈保持の確認)を実施済み

(Claude Opus 4.8)
```

### 例（1行で収まる場合）

```
コミットメッセージのスタイル規約をCLAUDE.mdに追加。 (Claude Sonnet 5)
```

軽微な変更（TASKS.mdの更新のみ等）では本文の箇条書きを省略し、タイトル＋モデル名のみでよい。
